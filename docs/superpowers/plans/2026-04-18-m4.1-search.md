# M4.1 — `sku search` (single-shard pinned) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `sku search` as a single-shard pinned command (`--provider` + `--service` required) that returns a filtered, sorted, limited list of SKU rows using the existing §4 output envelope. No binary release, no shard changes, no new dependencies. Cross-provider / multi-shard fan-out and `sku compare` remain deferred to M4.2.

**Architecture:**

1. **Catalog half (`internal/catalog/search.go`):** a new `Search(ctx, SearchFilter) ([]Row, error)` method composes a parameterised `SELECT` over `skus` + `resource_attrs` + a `prices`-aggregated sub-select (`SELECT sku_id, MIN(amount) AS min_price FROM prices GROUP BY sku_id`). Filters compose as `AND` predicates; `--sort` maps to an allow-list of columns; `--limit` caps the result. Predicate composition is hand-rolled — no `text/template`, no placeholder interpolation of user input (only bound `?` parameters), no reflection. Scanning reuses `lookupResource`'s field-by-field sql.NullX dance; the plan does **not** factor the scan path out in this milestone to keep the blast radius tiny.
2. **CLI half (`cmd/sku/search.go`):** a single new top-level `sku search` Cobra command under `root`. Requires `--provider` + `--service` (pins the shard via `catalog.ShardPath(<provider>-<service>)` exactly like the existing leaves). Supports `--kind`, `--region`, `--min-vcpu`, `--min-memory`, `--max-price`, `--sort`, `--limit`, and respects every global flag already on `root` (preset, jq, fields, pretty, dry-run, stale-ok, format). Dispatch mirrors `cmd/sku/aws_ec2.go` including the stale gate + missing-shard error path.
3. **Fixture (`internal/catalog/testdata/seed_search.sql`):** a ~20-row synthetic `aws-ec2`-shaped shard with instance types spanning 2-48 vCPU, 4-384 GB RAM, two regions, two kinds (so `--kind` has something to bite), and monotonically-increasing `compute` prices so sort/limit tests are deterministic.

**Tech Stack:** Go 1.25, `modernc.org/sqlite`, `github.com/spf13/cobra`. No new Go or Python deps.

**Spec ref:** `docs/superpowers/specs/2026-04-18-sku-design.md` §2 (data flow for search: `cmd/search -> internal/catalog.Query(filter) -> SQLite WHERE-driven scan`), §4 (search example + global flags + output envelope), §5 (indexes: `idx_skus_lookup`, `idx_resource_compute`; perf target `sku search` <40 ms p99 warm), §9 M4 entry.

**Explicit non-goals for m4.1** (declared to prevent scope creep into sibling M4.2 compare):
- Cross-provider / multi-shard fan-out. If `--provider` or `--service` is omitted, exit 4 (`validation`, `reason=flag_invalid`). M4.2 lifts this restriction as part of compare.
- `sku compare` command (cross-provider equivalence). M4.2.
- Region-group expansion (`--region us-east` → union of `us-east-1`, `us-east-2`, `eastus`, ...). M4.1 treats `--region` as the literal provider-region value. `pipeline/normalize/regions.yaml` is not touched in this plan.
- `--sort price.<dimension>` (per-dimension sort). `--sort price` in M4.1 sorts by per-SKU `MIN(amount)` across all dimensions, which is what the spec example `--sort price` already implies.
- Glob / prefix matching on `--resource-name` (SQLite `LIKE`). Not shipped; equality only, and only when `--resource-name` is passed.
- Full-text / fuzzy matching (`--q free text`). Not in spec for v1.
- `sku search --kind llm.text` querying the `openrouter` shard — LLM rows live in a cross-shard namespace (`openrouter` shard, no provider-service binary command), and `sku llm` already covers list/price. M4.1 explicitly rejects `--provider openrouter` so the pin rule stays consistent; LLM search arrives with M4.2 fan-out.
- Changes to `internal/output.Options` or preset projection. The existing `agent` preset renders a row correctly for search output; no new `search` preset.
- Benchmarks. The <40 ms p99 warm target in §5 is documented as an M1-established baseline for point lookups; a `BenchmarkSearch_*` harness lands with M4.2 once both compare and search share a fan-out path worth measuring.

```go
// internal/catalog/search.go
package catalog

import (
	"context"
	"fmt"
)

// SearchFilter captures the full set of filters `sku search` exposes in M4.1.
// All fields are optional except Provider + Service (which pin the shard).
// Zero values disable the corresponding predicate.
type SearchFilter struct {
	Provider     string
	Service      string
	Kind         string
	ResourceName string
	Region       string
	MinVCPU      int64
	MinMemoryGB  float64
	MaxPrice     float64 // 0 disables; negative is rejected by the caller
	Sort         string  // "", "price", "vcpu", "memory", "resource_name"
	Limit        int     // 0 disables (caller passes default)
}

// Search runs a generic filtered query over the shard's skus table. Unlike
// LookupVM / LookupDBRelational, Search does not require a resource_name or
// region and is free to return an empty slice. No-match is not an error —
// callers wrap empty results into skuerrors.NotFound at the command layer.
func (c *Catalog) Search(ctx context.Context, f SearchFilter) ([]Row, error) {
	if f.Provider == "" {
		return nil, fmt.Errorf("catalog: Search requires Provider")
	}
	if f.Service == "" {
		return nil, fmt.Errorf("catalog: Search requires Service")
	}
	_ = ctx
	return nil, nil
}
```

```go
func (c *Catalog) Search(ctx context.Context, f SearchFilter) ([]Row, error) {
	if f.Provider == "" {
		return nil, fmt.Errorf("catalog: Search requires Provider")
	}
	if f.Service == "" {
		return nil, fmt.Errorf("catalog: Search requires Service")
	}
	query, args := buildSearchQuery(f)
	rs, err := c.db.QueryContext(ctx, query, args...)
	if err != nil {
		return nil, fmt.Errorf("catalog: Search: %w", err)
	}
	defer func() { _ = rs.Close() }()

	var out []Row
	for rs.Next() {
		r, err := scanSearchRow(rs)
		if err != nil {
			return nil, err
		}
		r.CatalogVersion = c.catalogVersion
		r.Currency = c.currency
		if err := c.fillPrices(ctx, &r); err != nil {
			return nil, err
		}
		out = append(out, r)
	}
	return out, rs.Err()
}
```

Add the query builder + scanner in the same file:

```go
import (
	"database/sql"
	"encoding/json"
	"strings"
)

// buildSearchQuery composes the SELECT + WHERE + ORDER BY + LIMIT clauses
// from f. Only compile-time literals (never user input) are ever
// concatenated into the SQL text; every user-supplied value is bound as a ?.
func buildSearchQuery(f SearchFilter) (string, []any) {
	var where []string
	var args []any
	where = append(where, "s.provider = ?")
	args = append(args, f.Provider)
	where = append(where, "s.service = ?")
	args = append(args, f.Service)

	const base = `
SELECT s.sku_id, s.provider, s.service, s.kind, s.resource_name, s.region,
       s.region_normalized, s.terms_hash,
       t.commitment, t.tenancy, t.os, t.support_tier, t.upfront, t.payment_option,
       ra.vcpu, ra.memory_gb, ra.storage_gb, ra.gpu_count, ra.gpu_model,
       ra.architecture, ra.extra
FROM skus s
JOIN terms t ON t.sku_id = s.sku_id
LEFT JOIN resource_attrs ra ON ra.sku_id = s.sku_id
WHERE `
	query := base + strings.Join(where, " AND ") +
		"\nORDER BY s.resource_name, s.sku_id" //nolint:gosec // G202: no user input in SQL concatenation
	return query, args
}

func scanSearchRow(rs *sql.Rows) (Row, error) {
	var r Row
	var supportTier, upfront, paymentOption sql.NullString
	var vcpu sql.NullInt64
	var mem, storage sql.NullFloat64
	var gpuCount sql.NullInt64
	var gpuModel, arch, extraJSON sql.NullString
	if err := rs.Scan(
		&r.SKUID, &r.Provider, &r.Service, &r.Kind, &r.ResourceName, &r.Region,
		&r.RegionGroup, &r.TermsHash,
		&r.Terms.Commitment, &r.Terms.Tenancy, &r.Terms.OS,
		&supportTier, &upfront, &paymentOption,
		&vcpu, &mem, &storage, &gpuCount, &gpuModel, &arch, &extraJSON,
	); err != nil {
		return r, err
	}
	r.Terms.SupportTier = supportTier.String
	r.Terms.Upfront = upfront.String
	r.Terms.PaymentOption = paymentOption.String
	if vcpu.Valid {
		v := vcpu.Int64
		r.ResourceAttrs.VCPU = &v
	}
	if mem.Valid {
		v := mem.Float64
		r.ResourceAttrs.MemoryGB = &v
	}
	if storage.Valid {
		v := storage.Float64
		r.ResourceAttrs.StorageGB = &v
	}
	if gpuCount.Valid {
		v := gpuCount.Int64
		r.ResourceAttrs.GPUCount = &v
	}
	if gpuModel.Valid {
		v := gpuModel.String
		r.ResourceAttrs.GPUModel = &v
	}
	if arch.Valid {
		v := arch.String
		r.ResourceAttrs.Architecture = &v
	}
	if extraJSON.Valid && extraJSON.String != "" {
		_ = json.Unmarshal([]byte(extraJSON.String), &r.ResourceAttrs.Extra)
	}
	return r, nil
}
```
