# M3a.2 — AWS S3 + Lambda + EBS Shards (on-demand) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship three production-shaped AWS shards — `aws-s3` (kind `storage.object`), `aws-lambda` (kind `compute.function`), and `aws-ebs` (kind `storage.block`), on-demand only — end-to-end: DuckDB-based Python ingest from AWS Pricing API offer JSON (fixtures in tests, live URL path in ops), SQLite packaging via the existing shared packager, pure-Go catalog lookups for three new kinds, and six new Cobra leaves (`sku aws s3 {price,list}`, `sku aws lambda {price,list}`, `sku aws ebs {price,list}`) that return valid §4 envelopes. The inline `sku update` shard-source map is extended to accept the three new shard names.

**Architecture:**

1. **Pipeline half (`pipeline/ingest/aws_s3.py`, `pipeline/ingest/aws_lambda.py`, `pipeline/ingest/aws_ebs.py`):** each ingest module opens a DuckDB in-memory conn using the existing `ingest/_duckdb.py` helper, runs `read_json_auto` over the offer JSON (fixture or live `https://pricing.us-east-1.amazonaws.com/offers/v1.0/aws/<Amazon*>/current/index.json`), walks the nested `products` + `terms.OnDemand` shape in one SQL pass per price dimension, filters to the supported enum surface per kind, canonicalizes region via `pipeline/normalize/regions.yaml`, computes `terms_hash` via the `sku_terms_hash` DuckDB UDF, and emits NDJSON rows grouped so each row carries **all** price dimensions for its `(resource_name, region)` (S3 has three: storage + put/copy + get-select; Lambda has two: requests + duration; EBS has one in m3a.2 scope: storage).
2. **Packager (`pipeline/package/build_shard.py` — unchanged):** NDJSON rows → SQLite shard with the full §5 schema + `metadata.allowed_kinds` populated.
3. **Client half (pure Go):** `internal/catalog.LookupStorageObject`, `internal/catalog.LookupServerlessFunction`, `internal/catalog.LookupStorageBlock` reuse the existing `lookupResource` scan path introduced in m3a.1. `cmd/sku/aws_s3.go`, `cmd/sku/aws_lambda.go`, `cmd/sku/aws_ebs.go` register three sibling subcommand pairs under the existing `aws` parent from m3a.1 and reuse the `aws_helpers.go` helpers (`shardMissingErr`, `applyStaleGate`, `renderRows`). `cmd/sku/update.go`'s `shardSources` map gets three new entries.

**Tech Stack:** Python 3.11+, `duckdb>=0.10` (already a pipeline dep from m3a.1), existing `pipeline/package/build_shard.py` (unchanged), Go 1.25, `modernc.org/sqlite`, `github.com/spf13/cobra`. No new Go deps. No new Python deps.

**Spec ref:** `docs/superpowers/specs/2026-04-18-sku-design.md` §3 (format stack, shard granularity, update flow), §4 (command tree, global flags, output schema — `price` is always an array), §5 (kind taxonomy `storage.object` / `compute.function` / `storage.block`, terms_hash, `resource_attrs.durability_nines` + `availability_tier`, indexes), §6 (unit/integration test targets), §9 M3a entry.

**Explicit non-goals for m3a.2** (declared up front to prevent scope creep):
- `aws-dynamodb`, `aws-cloudfront` shards → m3a.3.
- `aws-*-commitments` shards → deferred past M3a per spec §3.
- Raw sibling shards (`aws-s3-raw.db.zst`, etc.) — no raw ingest until m3a.3 when the full updater module lands.
- `internal/updater` extraction with delta chain, manifest walking, ETag — m3a.3. `sku update` stays inline one-shot baseline per M1 pattern.
- `data-daily.yml` CI workflow — m3a.3.
- Validation harness (S3/Lambda/EBS cross-check vs upstream re-fetch) — m3a.3.
- `sku search` / `sku compare` — M4.

**Out-of-scope for m3a.2 within each shard:**
- **S3**: storage classes limited to `standard`, `standard-ia`, `one-zone-ia`, `intelligent-tiering`, `glacier-instant` (five classes that cover >90% of real usage). Glacier Flexible/Deep Archive, Reduced Redundancy, and Outposts classes → m3a.3. Data-transfer / egress dimensions → m3a.3 (different offer file shape, separate productFamily). Tiered storage pricing (the per-GB price that steps down above 50 TB for Standard) keeps the first tier only in m3a.2; all tiers ship in m3a.3 with the `prices.tier` column populated.
- **Lambda**: on-demand request + duration pricing only. Provisioned concurrency, ephemeral-storage surcharge, SnapStart → m3a.3. Architectures `x86_64` and `arm64`.
- **EBS**: storage dimension only (GB-Mo per volume type). Per-IOPS pricing (io1/io2/gp3 IOPS above baseline) and per-throughput pricing (gp3 throughput above 125 MBps, st1/sc1) → m3a.3. Volume types: `gp3`, `gp2`, `io2`, `st1`, `sc1` (five types cover >95% of real usage). Snapshots, archive tier → m3a.3.

Regions surface per shard: the five production regions established in m3a.1's `regions.yaml` seed (`us-east-1`, `us-east-2`, `us-west-2`, `eu-west-1`, `ap-northeast-1`). No new regions in this sub-plan.

| `productFamily` | Identifying attributes | Dimension emitted |
|---|---|---|
| `Storage` | `volumeType ∈ {Standard, Standard - Infrequent Access, One Zone - Infrequent Access, Intelligent-Tiering Frequent Access, Amazon Glacier Instant Retrieval}`, `storageClass = General Purpose` (or equivalent) | `storage` (unit: `GB-Mo`) |

```python
"""Normalize AWS S3 offer JSON into sku row dicts via DuckDB.

Spec §5 storage.object kind. S3 rows carry three price dimensions in m3a.2:
- storage (GB-Mo) from productFamily=Storage
- requests-put (requests) from productFamily=API Request, group=S3-API-Tier1
- requests-get (requests) from productFamily=API Request, group=S3-API-Tier2

We group by (storage_class, region) so each row has one entry per dimension.
A row missing any of the three dimensions is dropped with a stderr warning;
the fixture and the upstream AWS offer both publish all three for every
storage class + region in m3a.2's scope.
"""

from __future__ import annotations

import argparse
import sys
from pathlib import Path
from typing import Any, Iterable

from normalize.enums import apply_kind_defaults, load_enums
from normalize.terms import terms_hash

from ._duckdb import dumps, open_conn
from .aws_common import load_region_normalizer

_PROVIDER = "aws"
_SERVICE = "s3"
_KIND = "storage.object"

# Map the volumeType attribute string to our canonical storage-class slug.
# Classes outside this map are skipped (see m3a.2 non-goals: Glacier
# Flexible / Deep Archive / RRS / Outposts -> m3a.3).
_CLASS_MAP: dict[str, str] = {
    "Standard": "standard",
    "Standard - Infrequent Access": "standard-ia",
    "One Zone - Infrequent Access": "one-zone-ia",
    "Intelligent-Tiering Frequent Access": "intelligent-tiering",
    "Amazon Glacier Instant Retrieval": "glacier-instant",
}

# Durability + availability tier per class. Values from AWS public docs,
# encoded here to populate resource_attrs.
_CLASS_ATTRS: dict[str, dict[str, Any]] = {
    "standard":            {"durability_nines": 11, "availability_tier": "standard"},
    "standard-ia":         {"durability_nines": 11, "availability_tier": "infrequent"},
    "one-zone-ia":         {"durability_nines": 11, "availability_tier": "one-zone"},
    "intelligent-tiering": {"durability_nines": 11, "availability_tier": "standard"},
    "glacier-instant":     {"durability_nines": 11, "availability_tier": "archive"},
}

# Tier1 = PUT/COPY/POST/LIST, Tier2 = GET/SELECT. Map the `group` attribute
# to our price dimension slug.
_REQUEST_GROUP_MAP: dict[str, str] = {
    "S3-API-Tier1": "requests-put",
    "S3-API-Tier2": "requests-get",
}

# One SQL statement per product family because the filter predicates
# differ. Each returns: (sku, region, volume_type, dim_slug, usd, unit).
_STORAGE_SQL = """
WITH ondemand AS (
  SELECT sku_id, pd.unnest.value AS pd
  FROM (SELECT key AS sku_id, unnest(map_entries(value))
        FROM (SELECT unnest(map_entries(terms->'OnDemand')) FROM offer))
),
pd_min AS (
  -- Keep only the first tier (beginRange='0') per sku for m3a.2.
  SELECT sku_id,
         pd->>'unit' AS unit,
         CAST(pd->'pricePerUnit'->>'USD' AS DOUBLE) AS usd,
         pd->>'beginRange' AS begin_range
  FROM ondemand
)
SELECT
  p.sku AS sku_id,
  p.attributes->>'regionCode' AS region,
  p.attributes->>'volumeType' AS volume_type,
  'storage' AS dim_slug,
  f.usd, f.unit
FROM (SELECT unnest(map_entries(products)).value AS p FROM offer) pr
JOIN pd_min f ON f.sku_id = pr.p.sku
WHERE pr.p.productFamily = 'Storage'
  AND (f.begin_range IS NULL OR f.begin_range = '0')
"""

_REQUEST_SQL = """
WITH ondemand AS (
  SELECT sku_id, pd.unnest.value AS pd
  FROM (SELECT key AS sku_id, unnest(map_entries(value))
        FROM (SELECT unnest(map_entries(terms->'OnDemand')) FROM offer))
),
pd_min AS (
  SELECT sku_id,
         pd->>'unit' AS unit,
         CAST(pd->'pricePerUnit'->>'USD' AS DOUBLE) AS usd
  FROM ondemand
)
SELECT
  p.sku AS sku_id,
  p.attributes->>'regionCode' AS region,
  p.attributes->>'volumeType' AS volume_type,
  p.attributes->>'group' AS group_raw,
  f.usd, f.unit
FROM (SELECT unnest(map_entries(products)).value AS p FROM offer) pr
JOIN pd_min f ON f.sku_id = pr.p.sku
WHERE pr.p.productFamily = 'API Request'
"""


def ingest(*, offer_path: Path) -> Iterable[dict[str, Any]]:
    enums = load_enums()
    normalizer = load_region_normalizer()
    con = open_conn()
```

```python
"""Normalize AWS Lambda offer JSON into sku row dicts via DuckDB.

Spec §5 compute.function kind. Lambda rows carry two price dimensions:
- requests (unit: requests) from group=AWS-Lambda-Requests
- duration (unit: second) from group=AWS-Lambda-Duration

One row per (architecture, region). Provisioned concurrency, SnapStart, and
ephemeral-storage surcharges are out of scope for m3a.2 (see non-goals).
"""

from __future__ import annotations

import argparse
import sys
from pathlib import Path
from typing import Any, Iterable

from normalize.enums import apply_kind_defaults, load_enums
from normalize.terms import terms_hash

from ._duckdb import dumps, open_conn
from .aws_common import load_region_normalizer

_PROVIDER = "aws"
_SERVICE = "lambda"
_KIND = "compute.function"

_ARCH_MAP: dict[str, str] = {"x86": "x86_64", "arm": "arm64"}
_GROUP_MAP: dict[str, str] = {
    "AWS-Lambda-Requests": "requests",
    "AWS-Lambda-Duration": "duration",
}

_SQL = """
WITH ondemand AS (
  SELECT sku_id, pd.unnest.value AS pd
  FROM (SELECT key AS sku_id, unnest(map_entries(value))
        FROM (SELECT unnest(map_entries(terms->'OnDemand')) FROM offer))
),
pd_min AS (
  SELECT sku_id,
         pd->>'unit' AS unit,
         CAST(pd->'pricePerUnit'->>'USD' AS DOUBLE) AS usd
  FROM ondemand
)
SELECT
  p.sku AS sku_id,
  p.attributes->>'regionCode' AS region,
  p.attributes->>'archSupport' AS arch_raw,
  p.attributes->>'group' AS group_raw,
  f.usd, f.unit
FROM (SELECT unnest(map_entries(products)).value AS p FROM offer) pr
JOIN pd_min f ON f.sku_id = pr.p.sku
WHERE pr.p.productFamily = 'Serverless'
"""

    normalizer = load_region_normalizer()

    con.execute(f"CREATE VIEW offer AS SELECT * FROM read_json_auto('{offer_path}')")

    grouped: dict[tuple[str, str], dict[str, dict[str, Any]]] = {}
    for sku, region, arch_raw, group_raw, usd, unit in con.execute(_SQL).fetchall():

            "sku_id": sku_id,
            "provider": _PROVIDER,
            "service": _SERVICE,
            "kind": _KIND,
            "resource_name": arch,
            "region": region,
            "region_normalized": region_normalized,
            "terms_hash": terms_hash(terms),
            "resource_attrs": {
                "architecture": arch,
                "extra": {},
            },
            "terms": terms,
            "prices": [
                {"dimension": "requests", "tier": "", "amount": dims["requests"]["usd"], "unit": dims["requests"]["unit"].lower()},
                {"dimension": "duration", "tier": "", "amount": dims["duration"]["usd"], "unit": dims["duration"]["unit"].lower()},
            ],
        }
```

```go
// StorageObjectFilter captures the flags `sku aws s3 price/list` exposes.
// StorageClass is the user-facing name for resource_name on storage.object
// rows (e.g. "standard", "standard-ia"). Terms never carry tenancy/os for
// this kind — the client must zero-value those slots.
type StorageObjectFilter struct {
	Provider     string
	Service      string
	StorageClass string
	Region       string
	Terms        Terms
}

// ServerlessFunctionFilter captures the flags `sku aws lambda price/list` exposes.
// resource_name is the architecture string ("x86_64" / "arm64").
type ServerlessFunctionFilter struct {
	Provider     string
	Service      string
	Architecture string
	Region       string
	Terms        Terms
}

// StorageBlockFilter captures the flags `sku aws ebs price/list` exposes.
// resource_name is the volume-type slug ("gp3", "io2", ...).
type StorageBlockFilter struct {
	Provider   string
	Service    string
	VolumeType string
	Region     string
	Terms      Terms
}

// LookupStorageObject runs the storage.object point lookup / list query.
func (c *Catalog) LookupStorageObject(ctx context.Context, f StorageObjectFilter) ([]Row, error) {
	if f.StorageClass == "" {
		return nil, fmt.Errorf("catalog: LookupStorageObject requires StorageClass")
	}
	return c.lookupResource(ctx, "storage.object", f.Provider, f.Service,
		f.StorageClass, f.Region, f.Terms)
}

// LookupServerlessFunction runs the compute.function point lookup / list query.
func (c *Catalog) LookupServerlessFunction(ctx context.Context, f ServerlessFunctionFilter) ([]Row, error) {
	if f.Architecture == "" {
		return nil, fmt.Errorf("catalog: LookupServerlessFunction requires Architecture")
	}
	return c.lookupResource(ctx, "compute.function", f.Provider, f.Service,
		f.Architecture, f.Region, f.Terms)
}

// LookupStorageBlock runs the storage.block point lookup / list query.
func (c *Catalog) LookupStorageBlock(ctx context.Context, f StorageBlockFilter) ([]Row, error) {
	if f.VolumeType == "" {
		return nil, fmt.Errorf("catalog: LookupStorageBlock requires VolumeType")
	}
	return c.lookupResource(ctx, "storage.block", f.Provider, f.Service,
		f.VolumeType, f.Region, f.Terms)
}
```

```go
import (
```

```go
func newAWSS3Cmd() *cobra.Command {

type s3Flags struct {
	storageClass string
	region       string
	commitment   string
}

func (f *s3Flags) bind(c *cobra.Command) {
	c.Flags().StringVar(&f.storageClass, "storage-class", "",
		"standard | standard-ia | one-zone-ia | intelligent-tiering | glacier-instant")
	c.Flags().StringVar(&f.region, "region", "", "AWS region (e.g. us-east-1)")
	c.Flags().StringVar(&f.commitment, "commitment", "on_demand", "on_demand (only on-demand shipped in m3a.2)")
}

func (f *s3Flags) terms() catalog.Terms {
	return catalog.Terms{Commitment: f.commitment}
}

func newAWSS3PriceCmd() *cobra.Command {
	var f s3Flags
	c := &cobra.Command{
		Use:   "price",
		Short: "Price one S3 storage class",
		RunE:  func(cmd *cobra.Command, _ []string) error { return runAWSS3(cmd, &f, true) },
	}
	f.bind(c)
	return c
}

func newAWSS3ListCmd() *cobra.Command {
	var f s3Flags
	c := &cobra.Command{
		Use:   "list",
		Short: "List S3 storage-class SKUs matching filters (region optional)",
		RunE:  func(cmd *cobra.Command, _ []string) error { return runAWSS3(cmd, &f, false) },
	}
	f.bind(c)
	return c
}

func runAWSS3(cmd *cobra.Command, f *s3Flags, requireRegion bool) error {
	s := globalSettings(cmd)
	if f.storageClass == "" {
		e := skuerrors.Validation("flag_invalid", "storage-class", "",
			"pass --storage-class <class>, e.g. --storage-class standard")
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	if requireRegion && f.region == "" {
		e := skuerrors.Validation("flag_invalid", "region", "",
			"pass --region <aws-region>")
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	if s.DryRun {
		return output.EmitDryRun(cmd.OutOrStdout(), output.DryRunPlan{
			Command: "aws s3 " + cmd.Use,
			ResolvedArgs: map[string]any{
				"storage_class": f.storageClass,
				"region":        f.region,
				"commitment":    f.commitment,
			},
			Shards: []string{shardAWSS3},
			Preset: s.Preset,
		})
	}
	shardPath := catalog.ShardPath(shardAWSS3)
	if _, err := os.Stat(shardPath); err != nil {
		e := shardMissingErr(shardAWSS3)
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	cat, err := catalog.Open(shardPath)
	if err != nil {
		e := &skuerrors.E{Code: skuerrors.CodeServer, Message: err.Error()}
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	defer func() { _ = cat.Close() }()

	if stale := applyStaleGate(cmd, cat, shardAWSS3, s); stale != nil {
		return stale
	}

	rows, err := cat.LookupStorageObject(context.Background(), catalog.StorageObjectFilter{
		Provider: "aws", Service: "s3",
		StorageClass: f.storageClass,
		Region:       f.region,
		Terms:        f.terms(),
	})
	if err != nil {
		wrapped := fmt.Errorf("aws s3 %s: %w", cmd.Use, err)
		skuerrors.Write(cmd.ErrOrStderr(), wrapped)
		return wrapped
	}
	if len(rows) == 0 {
		e := skuerrors.NotFound("aws", "s3",
			map[string]any{
				"storage_class": f.storageClass,
				"region":        f.region,
			},
			"Try `sku schema aws s3` or drop --region for a list")
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	return renderRows(cmd, rows, s)
}
```

```go
func runAWSLambda(cmd *cobra.Command, f *lambdaFlags, requireRegion bool) error {
	s := globalSettings(cmd)
	if f.architecture == "" {
		e := skuerrors.Validation("flag_invalid", "architecture", "", "pass --architecture x86_64|arm64")
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	if requireRegion && f.region == "" {
		e := skuerrors.Validation("flag_invalid", "region", "", "pass --region <aws-region>")
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	if s.DryRun {
		return output.EmitDryRun(cmd.OutOrStdout(), output.DryRunPlan{
			Command: "aws lambda " + cmd.Use,
			ResolvedArgs: map[string]any{
				"architecture": f.architecture,
				"region":       f.region,
				"commitment":   f.commitment,
			},
			Shards: []string{shardAWSLambda},
			Preset: s.Preset,
		})
	}
	shardPath := catalog.ShardPath(shardAWSLambda)
	if _, err := os.Stat(shardPath); err != nil {
		e := shardMissingErr(shardAWSLambda)
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	cat, err := catalog.Open(shardPath)
	if err != nil {
		e := &skuerrors.E{Code: skuerrors.CodeServer, Message: err.Error()}
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	defer func() { _ = cat.Close() }()

	if stale := applyStaleGate(cmd, cat, shardAWSLambda, s); stale != nil {
		return stale
	}

	rows, err := cat.LookupServerlessFunction(context.Background(), catalog.ServerlessFunctionFilter{
		Provider: "aws", Service: "lambda",
		Architecture: f.architecture,
		Region:       f.region,
		Terms:        f.terms(),
	})
	if err != nil {
		wrapped := fmt.Errorf("aws lambda %s: %w", cmd.Use, err)
		skuerrors.Write(cmd.ErrOrStderr(), wrapped)
		return wrapped
	}
	if len(rows) == 0 {
		e := skuerrors.NotFound("aws", "lambda",
			map[string]any{"architecture": f.architecture, "region": f.region},
			"Try `sku schema aws lambda` or drop --region for a list")
		skuerrors.Write(cmd.ErrOrStderr(), e)
		return e
	}
	return renderRows(cmd, rows, s)
}
```
